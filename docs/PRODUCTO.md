# PRODUCTO

## Objetivo

La web no vende IA.

Debe lograr que un gerente piense:

```txt
esa es una pregunta que deberíamos poder responder
```

Y después:

```txt
cómo lo hace NexusG
```

## Dos páginas

### Qué

Responde:

```txt
qué problemas resuelve NexusG
```

No explica:

```txt
arquitectura
LLM
motores
MCP
integraciones internas
```

Las preguntas son la demostración.

### Cómo

Responde:

```txt
cómo trabaja NexusG sobre información existente
```

Debe aumentar confianza.

No debe convertirse en una página técnica.

## Dos trabajos

### Attention

```txt
saber dónde vale la pena dedicar atención
```

Manifestación visible:

```txt
PressureBoard
```

### Investigación

```txt
entender una situación compleja para decidir
```

Manifestación visible:

```txt
Workspace
```

## PressureBoard

No prioriza los problemas más grandes.

Prioriza las situaciones donde intervenir ahora puede cambiar más el resultado.

Toda card debe responder:

```txt
por qué esto merece atención ahora
```

## Workspace

No entrega análisis por mostrar análisis.

Entrega:

```txt
contexto preparado
evidencia
recomendación
```

Toda investigación debe cambiar una decisión.

## Regla de oro

```txt
el visitante debe pensar más en su empresa que en NexusG
```
